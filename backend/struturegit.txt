Etape 2: Pour le  Backend  et Frontend(Front end uniquement si on veux consommer les API backend 
sinon par défaut le front end à son propre environement)

Pour pouvoir demarrer les api backend il faut d'abord :

1-# Création de l’environnement
python3 -m venv .venv

2-# Activation (macOS / Linux)
source .venv/bin/activate

3-# Ou activation (Windows CMD)
.venv\Scripts\activate.bat

4-# Ou activation (Windows PowerShell)
.venv\Scripts\Activate.ps1

5-# Installation des dépendances
pip install -r requirements.txt

Etape 3 : Micro services 
1-Ce diriger vers le dossier backend et démarrer avec python la route avec la commande :
uvicorn auth.main:app --reload
Si on voulait démarrer le services apprenti il suffit de remplacer auth par apprenti  de meme pour les autres mico services 

2-Récuperer l'url de l'API  du mico service dans le fichier routes et le coler dans le navigateur  pour tester 
et voire comment la route focntionne les paramètre que elle prend en commpte avant de le consommer en front 
Exemple dans le fichier routes de Auth vous verrez : 
http://localhost:8005/auth/docs

3-Si vous aviez un message d'erreur c'est normale veillez regarder dans le termianle et reegarder le bon port surlequel tourne l'api démarrer en local sur votre machine

4-Si vous souhaitez démarrer tous les microservices veillez ouvire plusieur terminale et lancer a chaque fois :
uvicorn nom-microsservice.main:app --reload
